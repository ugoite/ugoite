# v0.2 Roadmap

`v0.2` is the next planned release stream after the foundational `v0.1` work.
Its focus is not to re-establish the core platform, but to expand how users
control views and how AI-assisted workflows grow beyond today's resource-first
MCP baseline inside trusted constraints.

**Current status:** `planned`

## Milestone Summary

| Milestone | Status | What it is expected to add |
|-----------|--------|----------------------------|
| User Controlled View | planned | Query-driven views, shareable view definitions, and user-owned presentation logic |
| AI-Enabled & AI-Used | planned | Broader MCP resource and tool coverage, workflow automation, vector search, voice transcription, and computational entry capabilities |

## What v0.2 Changes Compared with v0.1

`v0.1` focuses on making the base product durable: shared core logic, a stable
data model, SQL workflows, secure multi-user foundations, and a resource-first
MCP baseline. `v0.2` builds on that base by changing how users interact with
data and by expanding the AI surface beyond today's single-resource MCP
baseline.

### Planned additions in User Controlled View

- a view definition format stored with user data
- low-code composition between queries and UI components
- saved and shareable view definitions
- frontend support for query-driven panels

### Planned additions in AI-Enabled & AI-Used

- broader MCP resource and tool exposure beyond today's `entries/list` baseline
- AI workflow automation and batch processing
- vector search with configurable embedding providers
- voice memo transcription workflows
- computational entries with interactive code execution

## How to Read the v0.2 Tasks

The `v0.2` tracker is still planned rather than actively released. That means
the task lists in `docs/version/v0.2/*.yaml` describe intended capabilities and
testing expectations, not already-shipped behavior. When a milestone begins to
move into implementation, the version page should be updated to explain which
parts of the roadmap became real product behavior. Today the shipped MCP
contract remains one read-only resource (`ugoite://{space_id}/entries/list`)
with no tools or prompts.

## Upgrade Perspective

The most important conceptual upgrade from `v0.1` to `v0.2` is:

- `v0.1` makes the platform stable and operable
- `v0.2` makes the platform more expressive for users and broader in its
  AI-assisted workflows
