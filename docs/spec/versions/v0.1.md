# v0.1 Release Stream

`v0.1` is the foundational Ugoite release stream. It establishes the
local-first product baseline, then hardens that baseline with shared core
architecture, an Iceberg-backed data model, multi-user collaboration, and
release-readiness work.

**Current status:** `in_progress`

## Milestone Summary

| Milestone | Status | What it adds |
|-----------|--------|--------------|
| MVP | completed | Working local-first app, storage abstraction, REST/MCP baseline, frontend shell, Wasm sandbox, and critical-path tests |
| Full Configuration | completed | Shared `ugoite-core`, Form terminology, YAML requirements, and cross-module cleanup |
| Markdown as Table | completed | Iceberg-backed entry storage, Ugoite SQL, redesigned space UI, theming, and richer metadata |
| User Management | in progress | Authentication, authorization, memberships, service accounts, audit trails, and operator-facing auth UX |
| Release Preparation | planned | Deployable images, quick-start docs, and channel-scoped final release communication |

## What Changed Across v0.1

### Added in MVP

- Monorepo structure across backend, frontend, and CLI
- Space creation, entry CRUD, revision history, and integrity signing
- REST endpoints for spaces, entries, forms, assets, and search
- SolidStart frontend with the core local-first workflow
- MCP server endpoints and sandbox foundations

### Added in Full Configuration

- Shared Rust `ugoite-core` as the canonical logic layer
- Consistent "Form / Space / Entry / Asset" terminology across the repository
- YAML-based requirements and automated requirements-to-tests verification
- More explicit architecture boundaries between adapters and the core

### Added in Markdown as Table

- Iceberg-backed entry storage model
- Ugoite SQL for filter, sort, select, aggregate, and join workflows
- SQL metadata forms and stateless SQL session metadata
- New space UI, floating tabs, mobile responsiveness, and theme switching
- Sample data generation and richer metadata columns

### Still Being Added in User Management

`v0.1` is not only a "single drop" release. The stream still includes ongoing
user-management work:

- mandatory authentication across channels
- space membership lifecycle and role management
- audit event coverage
- CLI ergonomics for login, profile, and token management

At the time of writing, the design phase is complete and the implementation and
testing phases are largely advanced, but the milestone remains part of the
active `v0.1` stream until the release-preparation work is finished.

In `docs/version/v0.1.yaml`, each milestone is broken into phases and tasks.
